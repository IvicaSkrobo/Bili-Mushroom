<objective>
Fix two import workflow bugs:
1. Delete button in PostImportReviewDialog deletes the DB record but does not invalidate the TanStack Query cache, so the collection view still shows the "deleted" find after closing the review dialog.
2. The Rust `import_find` command unconditionally deletes source files after copying (line 236 of import.rs: `std::fs::remove_file`), ignoring the frontend's "delete source" checkbox. The checkbox only controls a redundant second deletion attempt in JS.

Purpose: Make delete-in-review actually remove finds from collection, and respect the user's choice to keep source files.
</objective>

<task type="auto">
  <name>Task 1: Fix Rust import_find to accept delete_source flag and stop unconditional file deletion</name>
  <files>src-tauri/src/commands/import.rs, src/lib/finds.ts, src/components/import/ImportDialog.tsx</files>
  <action>
**Root cause:** In `import.rs` line 234-236, after `std::fs::copy`, the code does `let _ = std::fs::remove_file(&payload.source_path)` unconditionally. Same pattern on line 298 for additional_photos. The frontend checkbox has no effect because the Rust command deletes first.

**Fix in `src-tauri/src/commands/import.rs`:**
1. Add a new parameter `delete_source: bool` to the `import_find` Tauri command signature (after `payloads`).
2. On line 236, wrap the `std::fs::remove_file` call in `if delete_source { ... }`. Change:
   ```rust
   let _ = std::fs::remove_file(&payload.source_path);
   ```
   to:
   ```rust
   if delete_source {
       let _ = std::fs::remove_file(&payload.source_path);
   }
   ```
3. Apply the same conditional on line 298 for additional_photos:
   ```rust
   if delete_source {
       let _ = std::fs::remove_file(additional_src);
   }
   ```
4. Keep the existing in-place detection (`is_already_in_storage`) — that path never deletes, so no change needed there.

**Fix in `src/lib/finds.ts`:**
Update the `importFind` function to pass `deleteSource`:
```typescript
export async function importFind(
  storagePath: string,
  payloads: ImportPayload[],
  deleteSource: boolean,
): Promise<ImportSummary> {
  return invoke<ImportSummary>('import_find', { storagePath, payloads, deleteSource });
}
```

**Fix in `src/components/import/ImportDialog.tsx`:**
1. In `handleImportAll`, pass `deleteSource` to the Rust command:
   Change line 281 from:
   ```typescript
   const summary = await importFind(storagePath, payloads);
   ```
   to:
   ```typescript
   const summary = await importFind(storagePath, payloads, deleteSource);
   ```
2. Remove the redundant JS-side file deletion block (lines 289-296 — the `if (deleteSource) { ... Promise.allSettled ... remove ... }` block). The Rust command now handles this correctly, and the JS `remove()` was a duplicate attempt anyway.

<task type="auto">
  <name>Task 2: Fix PostImportReviewDialog delete to invalidate query cache</name>
  <files>src/components/import/PostImportReviewDialog.tsx</files>
  <action>
**Root cause:** `handleDelete` in PostImportReviewDialog calls `deleteFind(storagePath, find.id, false)` and updates local `deletedIds` state, but never invalidates the TanStack Query cache. When the review dialog closes, the collection view still shows the deleted find from stale cache.

**Fix in `src/components/import/PostImportReviewDialog.tsx`:**
1. Import `useQueryClient` from `@tanstack/react-query` and `FINDS_QUERY_KEY` from `@/lib/finds`.
2. Get the query client inside the component: `const qc = useQueryClient();`
3. In `handleDelete`, after the successful `deleteFind` call and before `setDeletedIds`, add:
   ```typescript
   qc.invalidateQueries({ queryKey: [FINDS_QUERY_KEY, storagePath] });
   ```

This ensures the collection view refetches from DB (which no longer has the deleted record) when the user closes the review dialog or navigates to the collection.

Also pass `deleteFiles: true` instead of `false` to `deleteFind` so the copied photo files in the storage folder are also cleaned up (trashed). The find was just imported — keeping orphan files in storage with no DB record is wasteful. Change:
```typescript
await deleteFind(storagePath, find.id, false);
```
to:
```typescript
await deleteFind(storagePath, find.id, true);
```
  </action>

Run `npx tsc --noEmit` to confirm TypeScript compiles. Manually test: import photos, click delete on a find in review dialog, close dialog, verify find is gone from collection view.
  </verify>
  <done>
Deleting a find in the post-import review dialog removes it from the DB, trashes its copied files, and invalidates the query cache so the collection view is up-to-date when the dialog closes.
  </done>
