# Phase 260705-w54 Plan 01: Filled-field visual treatment in Import dialog Summary

Extracted the `filledClass` amber-border/bold-font "filled field" helper out of `CreateFindDialog.tsx` into a shared `src/lib/filledFieldStyle.ts` module, then applied it to all 9 shared-header fields in the Import photos dialog so typed values are visually distinguishable from placeholders, matching the New Find dialog.

## What Was Built

1. **`src/lib/filledFieldStyle.ts`** (new) — module-private style constants (`filledTextClass`, `filledUiTextClass`, `filledNumericClass`) plus exported `filledClass(value, variant)` function, copied verbatim from `CreateFindDialog.tsx`.
2. **`src/components/finds/CreateFindDialog.tsx`** — removed local `filledClass`/style-constant definitions, now imports `filledClass` from `@/lib/filledFieldStyle`. All 11 existing call sites unchanged; New Find dialog visual behavior is identical to before.
3. **`src/components/import/ImportDialog.tsx`** — added `filledClass` (and `cn`, previously missing) imports, then applied `filledClass` to all 9 shared-header fields: species name (`SpeciesNameEditor`), common name, date, country, region, location note (`LocationNoteInput` via `inputClassName`), observed count range, find notes, species/folder notes, and species description.
