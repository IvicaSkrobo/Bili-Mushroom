<objective>
Fix zone click resolution on the map so that when a user clicks a point where multiple zones (region and/or local) overlap in "all" view mode, the app shows a small on-map picker listing every zone at that point instead of always opening whichever zone happens to be on top of the Leaflet SVG stack.

Purpose: Zones layered in "all" mode currently make it impossible to select an underlying zone when it is fully covered by another zone's shape at the click point. The user must be able to choose which overlapping zone to open.

Output: A new `zonesContainingPoint` helper in `src/lib/zones.ts` (with a unit test), updated click handling in `src/components/map/ZoneLayers.tsx` that hit-tests all currently displayed zones and shows a `<Popup>` picker when 2+ match, and new hr/en i18n strings for the picker.
</objective>

// Existing precedent for the sorting convention to follow:
export function findContainingRegionZone(zones: Zone[], lat: number, lng: number): Zone | null {
  const matches = zones
    .filter((zone) => zone.zone_type === 'region')
    .filter((zone) => isPointInsideZone(lat, lng, zone))
    .sort((a, b) => zoneFootprint(a) - zoneFootprint(b));
  return matches[0] ?? null;
}
