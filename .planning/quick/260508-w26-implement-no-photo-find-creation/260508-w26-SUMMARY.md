## What Was Built

**Rust (finds.rs):**
- `CreateFindPayload` struct with all find fields (no source_path, no photos)
- `create_find` Tauri command: validates species_name, normalizes observed count range, inserts find row with zero photos, re-queries and returns FindRecord with photos: []
- `open_find_folder` photo-scope fallback: when scope=photo and find has no photos, falls back to species folder (created on demand) instead of returning an error

**TypeScript (finds.ts / useFinds.ts):**
- `CreateFindPayload` interface exported from finds.ts
- `createFind(storagePath, payload)` async function invoking `create_find`
- `useCreateFind()` mutation hook following the exact same pattern as `useUpdateFind`
- `create_find` default mock handler in tauri-mocks.ts

**UI (CreateFindDialog.tsx):**
- Mirrors EditFindDialog: SpeciesNameEditor, date, country, region, location mark, lat/lng, map picker, notes, observed count range
- Save disabled when species_name is empty or mutation is pending
- Form resets to blank on dialog open
- Error alert shown on mutation failure
- i18n keys: create.title ("New Find" / "Novi nalaz"), create.save ("Save" / "Spremi"), collection.newFind

**CollectionTab:**
- "New Find" outline button (SquarePen icon) placed before Import button
- CreateFindDialog mounted at JSX bottom, controlled by createFindOpen state

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] setup_in_memory_db missing migration 0012**
- **Found during:** Task 1
- **Issue:** `setup_in_memory_db` in `import.rs` test_helpers was missing migration 0012 (adds `observed_count_min` / `observed_count_max` columns). All existing tests in `finds.rs` that called `insert_find_row` were failing silently before because `make_find_record` sets those fields to `None` — but the `INSERT` statement references those column names, causing a schema error.
- **Fix:** Added `MIGRATION_0012` include and `conn.execute_batch` call to `setup_in_memory_db`
- **Files modified:** src-tauri/src/commands/import.rs
- **Commit:** bfc99a8

**2. [Rule 2 - Missing pub(crate)] normalize_observed_range not accessible cross-module**
- **Found during:** Task 1
- **Issue:** `normalize_observed_range` in import.rs is a private `fn`. `create_find` in finds.rs needed to call it.
- **Fix:** Added `normalize_observed_range_pub(crate)` alias in import.rs
- **Files modified:** src-tauri/src/commands/import.rs
- **Commit:** bfc99a8

**3. [Rule 1 - Bug] SpeciesNameEditor is contentEditable, not an input**
- **Found during:** Task 3 test run
- **Issue:** Tests used `screen.getByPlaceholderText` then `fireEvent.change` — both invalid for contentEditable divs. The editor uses `role="textbox"` with `aria-label={placeholder}` and fires `input` events.
- **Fix:** Changed to `screen.getByRole('textbox', { name: /species name/i })` and `speciesInput.textContent = '...'; fireEvent.input(speciesInput)`
- **Files modified:** CreateFindDialog.test.tsx
- **Commit:** b339d7a
