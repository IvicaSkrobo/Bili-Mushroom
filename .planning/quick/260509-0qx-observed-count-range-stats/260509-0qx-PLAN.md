# Observed-count range stats for species

## Goal

Surface per-find observed-count ranges as species-level aggregates: overall range (min–max) and average midpoint. Finds without observed counts are excluded from aggregates without breaking stats.

## Data model (existing)

`finds` table has:
- `observed_count` — exact count
- `observed_count_min` — low end of range
- `observed_count_max` — high end of range

## Files

1. `src-tauri/src/commands/stats.rs` — struct + sub-query + tests
2. `src/lib/stats.ts` — TS interface
3. `src/components/stats/SpeciesStatRow.tsx` — UI render

## Must-haves

- [ ] `SpeciesStatSummary` has `observed_min`, `observed_max`, `observed_avg`
- [ ] Sub-query uses `COALESCE(observed_count_min, observed_count)` for min/max
- [ ] AVG uses per-find midpoint when range available, exact count otherwise
- [ ] Finds with no obs data → all three fields NULL (not an error)
- [ ] 3 Rust tests pass
- [ ] TS interface updated
- [ ] UI shows range + avg only when data present
