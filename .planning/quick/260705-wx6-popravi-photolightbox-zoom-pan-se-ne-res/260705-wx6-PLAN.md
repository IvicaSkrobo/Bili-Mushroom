<objective>
Fix `PhotoLightbox.tsx` so that reopening the same photo (same `currentIndex` + same `fallbackFind?.id`/find) always resets zoom/pan/crop/rotation state to defaults, instead of preserving whatever zoom/pan the user left the photo at before closing.

Purpose: User reported (with screenshot) that reopening a previewed photo from Collection shows it zoomed-in/cropped instead of full-size. Root cause confirmed by direct code read: the reset `useEffect` at lines 87-97 of `PhotoLightbox.tsx` has dependency array `[currentIndex, fallbackFind?.id]`, which does not include `open`. The lightbox component stays mounted when closed (parent renders it with `open={false}`, not unmounted — confirmed via grep of `PhotoLightbox` usages in `SpeciesTab.tsx`/`CollectionTab.tsx`), so closing and reopening the same photo does not change `currentIndex` or `fallbackFind?.id`, and the effect never re-runs — `zoom`/`pan`/`cropMode`/`pendingRotation` retain their stale values from before close.

Output: `PhotoLightbox.tsx` reset effect fires on `open` transitioning true as well as on photo/find change (matching the reference pattern already used in the sibling `StagedPhotoViewer.tsx` from this same session, which resets on `[currentIndex, open]`). New regression test added in `PhotoLightbox.test.tsx` (no existing test file for this component — created fresh, following `StagedPhotoViewer.test.tsx` conventions).
</objective>
