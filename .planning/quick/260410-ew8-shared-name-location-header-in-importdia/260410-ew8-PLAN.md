4. Create a new inner component `LocateMeButton` that renders inside the MapContainer:
   ```tsx
   function LocateMeButton({ onLocate }: { onLocate: (latlng: L.LatLng) => void }) {
     const map = useMap();
     const [locating, setLocating] = useState(false);

     const handleLocate = () => {
       if (!navigator.geolocation) {
         toast.error('Geolocation not supported');
         return;
       }
       setLocating(true);
       navigator.geolocation.getCurrentPosition(
         (pos) => {
           const latlng = L.latLng(pos.coords.latitude, pos.coords.longitude);
           map.flyTo(latlng, 13);
           onLocate(latlng);
           setLocating(false);
         },
         (err) => {
           toast.error(`Location unavailable: ${err.message}`);
           setLocating(false);
         },
         { enableHighAccuracy: true, timeout: 10000 }
       );
     };

     return (
       <div className="absolute top-2 right-2 z-[1000]">
         <Button
           type="button"
           variant="secondary"
           size="icon"
           onClick={handleLocate}
           disabled={locating}
           aria-label="Use my location"
           title="Use my location"
           className="h-8 w-8 shadow-md"
         >
           <LocateFixed className={`h-4 w-4 ${locating ? 'animate-pulse' : ''}`} />
         </Button>
       </div>
     );
   }
   ```

5. Place `<LocateMeButton onLocate={setPin} />` inside the `MapContainer`, after `<ClickHandler>` and before the conditional `{pin && <Marker>}`.

In src-tauri/tauri.conf.json, add a `macOS` key inside the `bundle` object (alongside the existing `windows` key):

```json
"macOS": {
  "entitlements": "Entitlements.plist",
  "infoPlist": {
    "NSLocationWhenInUseUsageDescription": "Bili Mushroom uses your location to set the find location on the map."
  }
}
```

Then create a new file `src-tauri/Entitlements.plist`:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>com.apple.security.personal-information.location</key>
  <true/>
  <key>com.apple.security.app-sandbox</key>
  <true/>
  <key>com.apple.security.network.client</key>
  <true/>
</dict>
</plist>
```

The `com.apple.security.network.client` entitlement is needed because the app loads OSM map tiles over HTTPS. The `app-sandbox` key is required for macOS App Sandbox compatibility.

Also update the CSP in `app.security.csp` to allow geolocation by adding `https://*.tile.openstreetmap.org` to img-src (for map tiles loaded via HTTPS):
Current: `img-src 'self' asset: http://asset.localhost blob: data:`
Updated: `img-src 'self' asset: http://asset.localhost blob: data: https://*.tile.openstreetmap.org`

Note: `navigator.geolocation` works in WebView without extra CSP — it's a browser API. The entitlements file is what enables it on macOS.
