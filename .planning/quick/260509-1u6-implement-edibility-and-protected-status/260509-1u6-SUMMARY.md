# Phase quick-260509-1u6 Plan 01: Edibility and Protected Status Summary

Per-species edibility and protected_status metadata added to species_profiles table, surfaced via Forest Codex styled badge pair across all collection, map, and lightbox surfaces with a single edit entry in FolderEditDialog and prefill in ImportDialog.

## What Was Built

### Task 1 — DB migration + Rust model + upsert command extension
- Created `src-tauri/migrations/0013_species_profile_edibility.sql`: two `ALTER TABLE` statements adding `edibility TEXT` and `protected_status TEXT` as nullable columns; NULL = unknown per design
- Extended `SpeciesProfile` Rust struct with `edibility: Option<String>` and `protected_status: Option<String>`
- Extended `get_species_profiles` SELECT to fetch both new columns (index 3, 4)
- Extended `upsert_species_profile` command signature and SQL to INSERT + DO UPDATE both fields

**Commit:** b7daadf

### Task 2 — TS type + hook extension + shared badge infrastructure
- Extended `SpeciesProfile` TypeScript interface with `edibility?: string | null` and `protected_status?: string | null`
- Extended `upsertSpeciesProfile` function signature and Tauri invoke call to pass `edibility` and `protectedStatus` (Tauri auto-converts camelCase → snake_case)
- Extended `useUpsertSpeciesProfile` mutationFn to accept and forward both new fields
- Created `src/lib/speciesMetadata.ts`: enum literals, label maps, Forest Codex hex badge styles, `normalizeEdibility`/`normalizeProtectedStatus` guard functions
- Created `src/components/species/SpeciesMetadataBadges.tsx`: badge pair component with `size` ('sm'|'md') and `hideUnknown` props

**Commit:** 3c1f19b

### Task 3 — FolderEditDialog edibility/protected selects + ImportDialog prefill + CollectionTab wiring
- FolderEditDialog: added `speciesProfile` prop + `onSave` callback extended with `edibility`/`protectedStatus`; seeded state from prop in useEffect; added two `<select>` controls (Edibility, Protected Status) with Forest Codex dark styling; no internal upsert — values flow up via callback
- CollectionTab: built `speciesProfilesByName` useMemo Map; added `handleFolderSave` that merges edibility/protectedStatus into the single existing `upsertSpeciesProfile.mutate()` call; wired `speciesProfile` and `onSave` to `<FolderEditDialog>`
- ImportDialog: added `useSpeciesProfiles` + `useUpsertSpeciesProfile`; added `sharedEdibility`/`sharedProtectedStatus` state; extended prefill useEffect to seed from existing profile when species matches; added upsert on import completion when either value is non-unknown; added two compact `<select>` controls in form; reset in `resetState()`

**Commit:** 058f179

### Task 4 — Badge display across all surfaces
- FindCard: added `speciesProfile?: SpeciesProfile` prop; renders `SpeciesMetadataBadges` (size="sm", hideUnknown=true) below notes
- CollectionPins: added `useSpeciesProfiles` + `speciesProfilesByName` Map in `CollectionPinsInner`; added `speciesProfile?: SpeciesProfile` prop to `CollectionPopup`; renders badges (size="md", hideUnknown=true) below species name; passes `speciesProfilesByName.get(c.name)` from render loop
- PhotoLightbox: added `speciesProfile?: SpeciesProfile` prop; renders badges (size="md", hideUnknown=true) below species name in metadata panel
- CollectionTab folder header: renders badges (size="md", hideUnknown=false) so user always sees current values and knows they can edit
- CollectionTab inline find header row: renders badges (size="sm", hideUnknown=true) so badges only appear when species has edibility/protected set
- CollectionTab: passes `speciesProfile` to `<PhotoLightbox>` derived from `lightboxSpeciesName` or current photo's `species_name`

**Commit:** e7edddb
