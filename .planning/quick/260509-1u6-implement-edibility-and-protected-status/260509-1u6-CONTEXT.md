<decisions>
## Implementation Decisions

### Badge placement
- Show edibility and protected status badges in:
  - Collection folder header (always visible)
  - FindCard (compact badge inline with metadata)
  - Map popup (CollectionPopup in CollectionPins)
  - PhotoLightbox sidebar

### Edit entry point
- FolderEditDialog is the single editing home for both values
- Do NOT add primary controls to EditFindDialog (per-find dialog)
- Inline quick-edit in collection folder header is acceptable but secondary to FolderEditDialog

### Cross-import consistency
- When importing a species that already has a species profile with edibility/protected set, pre-fill those values in the import form
- User can change them before completing the import
- New species defaults gracefully to `unknown`/`unknown`

### Enum values (locked)
```
edibility: 'unknown' | 'edible' | 'inedible' | 'poisonous'
protected_status: 'unknown' | 'not_protected' | 'protected'
```

### Claude's Discretion
- Icon/badge visual design (keep simple, Forest Codex style, polish later)
- Whether to add edibility/protected to `useSpeciesProfiles` query or a separate hook
- Migration number (must be next in sequence after existing migrations)

</decisions>
