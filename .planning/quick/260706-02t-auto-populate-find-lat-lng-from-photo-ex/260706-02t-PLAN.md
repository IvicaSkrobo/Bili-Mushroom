<objective>
During photo import, automatically populate a find's lat/lng from the first photo's GPS EXIF data — but only when the find does not already have coordinates. Manual user input (via `EditFindDialog`'s lat/lng fields or `LocationPickerMap`) must always take precedence and must never be silently overwritten by EXIF data pulled in later.

This applies to two existing code paths:
1. **New find creation during import** — `import_find` in `src-tauri/src/commands/import.rs`.
2. **Adding photos to an existing find** — `add_find_photos` in `src-tauri/src/commands/finds.rs`.

Purpose: Foragers currently must manually enter GPS coordinates for every find even though their phone photos already carry this data in EXIF. Auto-populating removes that friction while preserving full manual control.

Output: Shared helpers (`first_gps_coords_from_paths`, `first_gps_coords_from_staged`, `resolve_find_coords`) in `import.rs`, wired into both `import_find`'s existing transactional copy/commit/delete flow and `add_find_photos`'s existing copy/insert flow. Rust unit tests covering the three required scenarios (sets when null, does not overwrite when set, no GPS leaves null).
</objective>
