<objective>
Fix the multi-species hover tooltip on the map so species names stack one per line (vertically) instead of rendering in one row.

Purpose: When multiple mushroom finds share the exact same coordinate, hovering the pin currently shows all species names in a row that visually collapses/overlaps, hurting readability. The species list itself is already correct (a prior fix resolved a bug where only the last-entered species showed) — only the CSS/layout of the tooltip needs to change.

Output: Updated `CollectionPins.tsx` tooltip markup using explicit flex-column stacking, and an updated `.bili-species-tooltip` CSS rule in `src/index.css` that no longer suppresses vertical stacking via `white-space: nowrap`, while preserving all existing dark Forest Codex visual styling (background, border, radius, shadow, colors).
</objective>

```css
}
```

Root cause: Leaflet's base `.leaflet-tooltip` rule (`node_modules/leaflet/dist/leaflet.css`) sets `white-space: nowrap` and auto-sizes the tooltip box to content width/height without an explicit block-stacking context. Combined with the app's own `white-space: nowrap` on `.bili-species-tooltip`, the per-species `<div>` rows do not stack cleanly. Fix must be explicit (flex column) rather than relying on implicit `<div>` block stacking, and must stop overriding `white-space` to `nowrap` at the container level.
</interfaces>

Do not touch `locationGroupsFromFinds` (species grouping logic) or `labelText` generation for the pin itself — this is purely a CSS/layout fix for the hover tooltip. Do not modify any other files.
