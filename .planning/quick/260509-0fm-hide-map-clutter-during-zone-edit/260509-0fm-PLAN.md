# Hide map clutter during zone polygon editing

## Goal

When a polygon editor is active (drawing or editing a local or region zone), hide unrelated map elements so the user can focus on the active boundary. Restore normal visibility on save or cancel.

## Scope

Single file: `src/components/map/FindsMap.tsx`

No other files change.

## Tasks

### T1 — Add useMemo import

`useMemo` not currently in React import line. Add it.

File: `src/components/map/FindsMap.tsx` line 3

### T2 — Compute focusFinds

Inside `FindsMap` function body, after props destructuring, add:

```ts
const focusFinds = useMemo(() => {
  if (!polygonEditorActive) return finds;
  if (polygonEditorZoneType === 'local' && drawTargetFind != null) {
    return finds.filter((f) => f.id === drawTargetFind.id);
  }
  if (polygonEditorZoneType === 'region' && drawTargetFind != null) {
    return finds.filter((f) => f.species_name === drawTargetFind.species_name);
  }
  return [];
}, [polygonEditorActive, polygonEditorZoneType, drawTargetFind, finds]);
```

### T3 — Hide all zones during editing

Change `hiddenZoneIds` prop on `<ZoneLayers>`:

Before:
```tsx
hiddenZoneIds={polygonEditorActive && activeZoneId != null ? [activeZoneId] : []}
```

After:
```tsx
hiddenZoneIds={polygonEditorActive ? zones.map((z) => z.id) : []}
```

When editing, all static zone overlays are hidden. The draft polygon (PolygonDraftLayer / PolygonEditHandles) shows the active boundary.

### T4 — Show filtered pins during editing

Change `CollectionPins` render block:

Before:
```tsx
{!focusMode && (
  <CollectionPins
    finds={finds}
    ...
  />
)}
```

After (remove focusMode gate, pass focusFinds):
```tsx
<CollectionPins
  finds={focusFinds}
  ...
/>
```

`focusFinds` is `[]` when no drawTargetFind is set (edge case), so no pins render. OnlineStatusBadge `!focusMode` gate is unchanged.
