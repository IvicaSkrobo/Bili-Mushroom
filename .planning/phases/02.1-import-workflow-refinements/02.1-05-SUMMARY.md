# Phase 02.1 Plan 05: PostImportReviewDialog and BulkMetadataBar Summary

**One-liner:** Post-import review dialog showing imported/skipped summary with inline edit + bulk metadata bar for applying shared fields across batch imports.

## What Was Built

### Task 1: PostImportReviewDialog (TDD)

`src/components/import/PostImportReviewDialog.tsx` — Dialog that opens after every successful import showing the `ImportSummary` result. Features:
- Opens when `summary !== null`, closes when `onOpenChange(false)` called
- Lists all imported finds with species name (or filename fallback), date, and an Edit button
- Edit button opens an inline `EditFindDialog` for the selected find without leaving the review
- Shows a separate skipped section only when `skipped.length > 0`
- Done button calls `onOpenChange(false)` to signal parent to clear state and close ImportDialog

8 TDD tests covering: null state, open state, imported count display, species name rendering, skipped section presence/absence, Done button, Edit button flow.

### Task 2: BulkMetadataBar (TDD)

`src/components/import/BulkMetadataBar.tsx` — Bar with shared Species name, Date, and Country inputs plus an "Apply to all" button. Features:
- Renders only when `itemCount >= 2` (returns null for 0 or 1)
- Apply to all collects only non-empty field values into a `BulkFields` partial object
- Calls `onApplyAll(fields)` — parent maps over pending items and applies non-undefined fields
- All inputs reset to empty after applying

7 TDD tests covering: visibility conditions, field presence, apply-to-all with values, apply-to-all with empty values, input reset behavior.

### Task 3: ImportDialog Wiring

`src/components/import/ImportDialog.tsx` updated:
- Added imports for `PostImportReviewDialog`, `BulkMetadataBar`, `BulkFields`, `ImportSummary`
- Added `importSummary` and `reviewOpen` state
- `handleImportAll` no longer calls `setPending([])` or `onOpenChange(false)` — instead stores summary and sets `reviewOpen = true`
- `handleReviewClose` resets all shared state (name, date, country, region, locationNote, folderNotes, location, importSummary, reviewOpen) then calls `onOpenChange(false)`
- `handleApplyAll` maps over pending items applying only the non-undefined `BulkFields`
- `BulkMetadataBar` rendered above the preview list inside the `pending.length > 0` block
- `PostImportReviewDialog` rendered after closing `</Dialog>` tag (sibling in fragment)

`src/components/import/ImportDialog.test.tsx` updated:
- Fixed `useAppStore` mock to include `language: 'en'` (was missing, causing `useT()` to fail)
- Fixed `sampleSummary` to use correct `Find` shape (`photos: []` instead of `photo_path`)
- Added mocks for `LocationPickerMap` and `EditFindDialog` (dependency isolation)
- Updated "on successful import" test: now checks PostImportReviewDialog opens instead of toast + onOpenChange(false)
- Fixed "folder picker filters" test: counts remove buttons instead of species inputs (BulkMetadataBar adds a species input when pending >= 2)
- Added species name fill before Import All click in tests that need the button enabled
