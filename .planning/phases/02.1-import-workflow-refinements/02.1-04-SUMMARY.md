## What Was Built

### FindPreviewCard.tsx
- Added `useState(false)` for `mapOpen` dialog control
- Added `handleMapConfirm(lat, lng)` callback that calls `onChange({ ...payload, lat, lng })`
- Restructured the lat/lng input row from a `grid-cols-2` nested div to a flat `col-span-2 grid grid-cols-[1fr_1fr_auto]` to accommodate the MapPin button
- Added `MapPin` icon button with `aria-label="Pick on map"` that sets `mapOpen(true)`
- Renders `LocationPickerMap` as a sibling to `Card` (wrapped in a fragment) — placing a Dialog inside a Card caused nesting issues
- Passes `initialLatLng` derived from payload when both lat and lng are non-null, else passes `null`

### ImportDialog.tsx
- Added `additional_photos: []` to `buildInitialPayload` so the returned object satisfies the `ImportPayload` TypeScript type (which already had the field in `finds.ts`)

### FindPreviewCard.test.tsx
- Added `vi.mock('@/components/map/LocationPickerMap', ...)` at module level (before imports) to satisfy Vitest's hoisting requirement
- Updated `basePayload` to include `additional_photos: []`
- Added 5 new tests: button visible, map opens on click, onConfirm writes lat/lng, initialLatLng passed with coordinates, initialLatLng=null when no coordinates
