# Phase 06-03 Plan: Collection Scalability

## Goal

Keep the Collection tab responsive with thousands of finds by virtualizing/paginating what is visible.

## Scope

- Virtualized species/folder list.
- Paginated or virtualized expanded find rows.
- On-demand photo thumbnails for visible rows only.
- Search/date/favorite filters backed by summary queries where possible.

## Tasks

1. Choose a small virtualization approach.
   - Prefer a focused library such as TanStack Virtual if it fits the app.
   - Avoid broad UI rewrites.

2. Split collection data needs.
   - Species list uses summary query.
   - Expanded species loads its find page(s).
   - Expanded find loads photo grid on demand.

3. Preserve existing workflows.
   - Edit/delete/favorite.
   - Folder notes/profile edits.
   - Open on map/species/folder.
   - Lightbox from visible rows.

4. Tune interactions for large folders.
   - Page size or "load more" for thousands of finds under one species.
   - Stable scroll position after updates.
   - Targeted query invalidation after edit/delete/import.

## Success Criteria

- Collection tab does not render all finds or all photo thumbnails at once.
- Search and date filters remain usable on large libraries.
- Editing/deleting one find does not require rebuilding the whole collection in the UI.

## Validation

- Add synthetic 10k-find render test or benchmark.
- Verify row heights/text fit across dark/light themes.
- Smoke-test import -> collection -> edit/delete -> stats invalidation.
