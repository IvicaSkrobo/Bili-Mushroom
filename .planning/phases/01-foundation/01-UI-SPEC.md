# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (CLI-managed, not yet initialized — executor must run `npx shadcn init` before implementing) |
| Preset | none — use shadcn default "New York" style, "neutral" base color, CSS variables ON |
| Component library | Radix UI (via shadcn — do not use raw Radix directly) |
| Icon library | lucide-react (bundled with shadcn/ui) |
| Font | System font stack: `ui-sans-serif, system-ui, -apple-system, sans-serif` — no web font download; Windows desktop app ships without guaranteed internet |

**Init command for executor:**
```
npx shadcn@latest init
```
When prompted: Style = New York, Base color = Neutral, CSS variables = Yes.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, tight inline padding |
| sm | 8px | Button internal padding (vertical), list item gaps |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, dialog content padding |
| xl | 32px | Layout gaps between major UI regions |
| 2xl | 48px | Empty state vertical centering padding |
| 3xl | 64px | Page-level top/bottom breathing room |

Exceptions:
- Tab bar height: 44px (minimum touch/click target on Windows; satisfies accessibility)
- First-run dialog width: 480px fixed (not a spacing token — a layout constraint)

## Typography

System font stack is used throughout — no custom font import.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.15 |

Notes:
- Body (14px) is the default for all paragraph text, list content, and dialog prose.
- Label (12px) is used for tab labels, form field labels, metadata captions, and secondary descriptions.
- Heading (20px) is used for dialog titles, section headings, and empty-state headings.
- Display (28px) is reserved for the app window title area and first-run onboarding headline only — not used on tabs or dialogs.
- No italic or bold-italic styles in Phase 1.

## Color

shadcn/ui "neutral" base provides the palette. All values reference shadcn CSS variables — do not hardcode hex values in component code.

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background` (approx #fafafa in light) | Main window background, tab content area |
| Secondary (30%) | `--card` / `--muted` (approx #f4f4f5 in light) | Tab bar surface, sidebar/nav area, dialog backdrop overlay region |
| Accent (10%) | `--primary` — forest green: `oklch(0.45 0.14 145)` | See reserved list below |
| Destructive | `--destructive` (shadcn default red) | Destructive confirmation actions only |

**Accent reserved for:**
1. The active/selected tab indicator underline or highlight
2. The primary CTA button in the first-run onboarding dialog ("Choose Folder")
3. The "Change Folder" button in the Settings panel (primary action only)
4. Focus rings on interactive elements (inherits from shadcn's ring variables)

**Accent is NOT used for:** empty state illustrations, placeholder text, tab labels (unselected), decorative elements.

**Theme mode:** Light mode only for Phase 1. Dark mode is not in scope. Do not implement dark mode toggle.

**To apply the forest green accent**, set in `app/globals.css` (or equivalent):
```css
:root {
  --primary: oklch(0.45 0.14 145);
  --primary-foreground: oklch(0.98 0 0);
  --ring: oklch(0.45 0.14 145);
}
```

## Layout Contract

### Window Shell

```
┌─────────────────────────────────────────────────┐
│  [Bili Mushroom]                    [Settings ⚙] │  ← Title bar / toolbar row — 48px height
├─────────────────────────────────────────────────┤
│ [Collection] [Map] [Species] [Browse] [Stats]   │  ← Tab bar — 44px height, full width
├─────────────────────────────────────────────────┤
│                                                 │
│             Tab content area                    │  ← Fills remaining height
│          (placeholder / empty states)           │
│                                                 │
└─────────────────────────────────────────────────┘
```

- Tab navigation: **top horizontal tabs** (not sidebar). Windows desktop convention; shadcn `<Tabs>` component.
- Active tab: accent-colored bottom border (2px), semibold label weight.
- Inactive tabs: muted label color, regular weight.
- Settings gear icon (`lucide-react: Settings`) in the top-right of the title bar row — triggers Settings panel or dialog (Phase 1 only shows "Change Folder" option).

### First-Run Onboarding Dialog

- Trigger: on app launch when no storage folder is configured in Tauri app data dir.
- Modal: true (cannot be dismissed by clicking outside or pressing Escape — D-03: must choose a folder).
- Width: 480px. Height: auto (content-driven, min 320px).
- No close [X] button — the dialog is a hard gate.

Content layout:
```
┌──────────────────────────────┐
│  [Mushroom icon — 32px]       │  ← lucide-react: Leaf or Sprout icon, accent color
│  Choose Your Mushroom Library │  ← Display (28px, semibold)
│  ─────────────────────────── │  ← Separator
│  [Body text — explanation]    │  ← Body (14px, regular)
│                               │
│  [Selected path or empty]     │  ← Muted text field showing chosen path
│                               │
│  [Choose Folder]              │  ← Primary Button (accent), full width
└──────────────────────────────┘
```

Icon choice: `Sprout` from lucide-react (represents growth/nature, 32px, accent color).

### Empty States (per tab)

All tabs in Phase 1 show a centered empty state. Each uses the same layout:
- Icon: 48px lucide-react icon, muted foreground color (not accent).
- Heading: 20px semibold.
- Body: 14px regular, muted, max-width 320px, centered.
- No CTA button in empty states for Phase 1 (features come in later phases).

| Tab | Icon | Heading | Body |
|-----|------|---------|------|
| Collection | `GalleryHorizontal` | Your collection is empty | Import your first mushroom find to get started. |
| Map | `MapPin` | No finds on the map yet | Your mushroom finds will appear here once you start importing. |
| Species | `BookOpen` | Species database | The built-in species guide will be available here. |
| Browse | `Search` | Nothing to browse yet | Start building your collection to browse and filter your finds. |
| Stats | `BarChart3` | No stats yet | Your foraging stats will appear here once you have finds in your collection. |

### Migration Failure Error Dialog

- Trigger: on startup when a database migration fails.
- Modal: true. Cannot be dismissed — app refuses to start (D-06).
- No close [X] button.
- Width: 480px.

Content:
```
┌──────────────────────────────┐
│  [Alert triangle icon]        │  ← lucide-react: AlertTriangle, destructive color, 32px
│  Database Error               │  ← Heading (20px, semibold)
│  ─────────────────────────── │
│  [Error details — technical]  │  ← Body (14px, monospace for error string)
│  [Explanation — plain English]│  ← Body (14px, regular)
│                               │
│  [Quit App]   [Open Log File] │  ← Two buttons: default Quit (destructive), secondary Open Log
└──────────────────────────────┘
```

### Settings Panel

Accessible via the Settings gear icon in the title bar. Implement as a Dialog (not a sidebar) for Phase 1 — only one setting exists.

Content:
- Heading: "Settings" (20px, semibold)
- Single section: "Storage Folder"
  - Label: "Mushroom Library Location" (12px, medium)
  - Displayed path: current folder path (14px, muted, monospace)
  - Button: "Change Folder" (secondary variant — not accent primary, to distinguish from first-run CTA)
  - Alert below button: "Changing your library folder will ask whether to move or copy your existing data." (shadcn `<Alert>` with `Info` icon)

## Interaction Contract

### Keyboard Navigation
- All interactive elements reachable by Tab key (Radix UI primitives handle this).
- Active tab switchable with Left/Right arrow keys (Radix `<Tabs>` default behavior).
- First-run dialog: Tab cycles between "Choose Folder" button and any other focusable elements. Enter activates focused button.
- Migration error dialog: Tab cycles Quit/Open Log. Enter activates focused button.
- No custom keyboard shortcuts in Phase 1.

### Focus Management
- On first-run dialog open: focus lands on "Choose Folder" button.
- On migration error dialog open: focus lands on "Quit App" button (primary destructive action).
- On settings dialog open: focus lands on "Change Folder" button.

### Loading States
- No loading spinners needed in Phase 1. The only async operation is the OS folder picker (native dialog — no custom loading state needed). Migration runs at startup before the window is shown — no in-app loading state required.

### Window Sizing
- Minimum window size: 900px wide × 600px tall.
- Initial window size: 1200px × 800px.
- Resizable: yes.
- These constraints are set in `tauri.conf.json` — not a CSS concern.
