## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, dot indicator margin |
| sm | 8px | Compact popup row spacing, inline padding |
| md | 16px | Default element spacing, popup content padding |
| lg | 24px | Section padding, modal content padding |
| xl | 32px | Layout gaps, modal header spacing |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not applicable on map tab) |

Exceptions:
- Map container: `height: 100%` — fills available tab viewport, no fixed pixel height
- Online/offline dot: 10px diameter — intentionally sub-grid, must not draw attention
- Cluster badge: 24px min-width, 24px height — touch/click target override for dense pin areas

## Typography

Exactly 2 weights declared: **400 (regular)** and **600 (semibold)**.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | DM Sans | 14px | 400 | 1.5 | Popup metadata rows, Settings text, modal body copy |
| Label | DM Sans | 12px | 400 | 1.4 | Badge text, cluster count, layer switcher labels, coordinates display |
| Heading | Playfair Display | 16px italic | 600 | 1.2 | Species name in Level 1 popup, Level 2 mini card heading |
| Display | DM Sans | 20px | 600 | 1.2 | Location picker modal title ("Pick a location") |

Notes:
- Coordinates (`lat`, `lng`) always use JetBrains Mono, 12px weight 400 — must be monospaced for alignment
- Date strings use DM Sans 12px weight 400 (body small)
- Cluster count uses DM Sans 12px weight 400 — visual distinction from surrounding content is achieved via the amber filled circle background, not weight; 400 on a 24px amber circle at 12px is sufficient

## Color

All values reference existing CSS variables in `src/index.css`. Dark mode (Forest Codex) is the only mode; dark class is always applied.

| Role | CSS Variable | Dark value (oklch) | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `--background` | `oklch(0.13 0.015 135)` — deep moss | Map tab background, tile area behind legend |
| Secondary (30%) | `--card` / `--secondary` | `oklch(0.18 0.02 132)` — dark forest card | Popup background, mini card, modal backdrop panel, Settings rows |
| Accent (10%) | `--primary` | `oklch(0.80 0.14 78)` — chanterelle amber | Active cluster badge background, "Confirm location" button, Edit button hover, pin SVG fill, tab active underline |
| Destructive | `--destructive` | `oklch(0.60 0.20 25)` — deep red | "Clear tile cache" button only |
| Online dot | literal `#4ade80` (green-400) | — | Online status indicator dot ONLY |
| Offline dot | `--muted-foreground` | `oklch(0.62 0.04 125)` | Offline/cached status indicator dot ONLY |
| Border | `--border` | `oklch(0.30 0.025 130)` | Popup border, card border, modal border |
| Muted text | `--muted-foreground` | `oklch(0.62 0.04 125)` | Date rows, secondary metadata, coordinates label text |

Accent reserved for ONLY:
1. Cluster badge background (filled amber circle with dark count text)
2. Primary action buttons: "Confirm location" (picker modal), "Edit find" (Level 2 popup)
3. Find pin marker SVG fill color
4. Tab active underline (existing global rule — do not override)

**Primary visual focal point:** Amber find pins are the primary visual anchor against the dark moss tile background. All other map chrome (zoom controls, layer switcher, attribution) is styled with dark card tones to recede. Pins compete with nothing.

## Interaction Contract

### Map Tab (MapTab.tsx)
- Full-height layout: map fills the entire tab viewport (`height: 100%`, `width: 100%`)
- Leaflet CSS imported globally in `main.tsx` (`import 'leaflet/dist/leaflet.css'`)
- Default center: `[45.1, 15.2]`, zoom 7 (Croatia/Balkans region)
- When finds with coordinates are present: `map.fitBounds()` called via `useMap()` hook on initial load
- `animate-fade-up` applied to the map container wrapper div on mount (consistent with tab transition pattern)

### Find Pins
- One pin per unique lat/lng coordinate group
- Single find at coordinate: standard Leaflet marker with amber SVG fill
- Multiple finds at same lat/lng: cluster pin — amber circle badge with dark count text (e.g. "3"), 24px diameter, weight 400
- Pin click opens Level 1 popup immediately (no hover states — click only; desktop app)

### Pin Popup: Level 1
- Width: 200px fixed
- Content: species name (Playfair Display 16px italic 600) + date found (DM Sans 12px 400 muted)
- Cluster Level 1: scrollable list of all finds at that spot — each row: species name + date, max-height 200px with overflow-y scroll
- Each name in cluster list is clickable → expands to Level 2
- Species name in single-find popup is clickable → expands to Level 2

### Pin Popup: Level 2
- Replaces Level 1 content (not a second popup — inline expansion within same Leaflet Popup)
- Width: 240px fixed
- Content: thumbnail (48x48px, rounded-sm, object-cover) + species name (heading) + date (body small) + "Edit find" button (ghost variant, amber on hover)
- "Edit find" button: closes popup, sets activeTab to "collection", opens EditFindDialog for that find ID
- Back chevron (lucide `ChevronLeft`, 16px) at top-left of Level 2 collapses back to Level 1

### Location Picker Modal
- Trigger: "Pick on map" button in EditFindDialog (ghost variant, MapPin icon)
- Dialog: `fullscreen` behavior — `max-w-none w-screen h-screen` on DialogContent
- Modal header: "Pick a location" (Display 20px 600) + close X button
- Map fills remaining height after header (use `calc(100vh - 64px)`)
- Default behavior: if find has existing lat/lng → map centers there at zoom 13 with pre-placed moveable pin; if no coords → center on Croatia `[45.1, 15.2]` zoom 7, no pin until user clicks
- Click anywhere on map: places or moves a single draggable pin
- Dragging pin: pin moves; coordinates preview updates in real time below the map — display as `45.123456, 14.456789` in JetBrains Mono 12px
- "Confirm location" button (primary/accent): fixed at bottom of modal, full-width, calls `onLocationPicked(lat, lng)` and closes modal
- Confirmation button is disabled until a pin has been placed

### Online/Offline Status Badge
- Position: `leaflet-bottom leaflet-left` via custom Leaflet control (not a React overlay div)
- Appearance: 10px circle dot + 8px DM Sans label ("Online" / "Cached")
- Green `#4ade80` dot when fetching live tiles; muted-foreground dot when serving from cache
- No animation, no pulse — deliberately unobtrusive

### Settings Panel (SettingsDialog additions)
- New section heading: "Map Cache" (DM Sans 14px weight 400)
- Row 1: "Tile cache size" label + right-aligned current size display (e.g. "42 MB") in muted-foreground
- Row 2: "Max cache size" label + Input field (width 80px) + "MB" suffix label
- Row 3: "Clear tile cache" button (destructive variant, outline style) — triggers AlertDialog for confirmation
- AlertDialog confirmation copy: "This will delete all cached map tiles. Areas you've previously viewed will need to reload when you're next online." — actions: "Cancel" (default) + "Clear cache" (destructive)

## Map-Specific Visual Rules

### Leaflet Layer Styling Override
The default Leaflet attribution control and zoom control use light styles. Override with Forest Codex tokens:

```css
/* In src/index.css — add after existing rules */
.leaflet-control-attribution {
  background: var(--card) !important;
  color: var(--muted-foreground) !important;
  font-family: var(--font-sans) !important;
  font-size: 10px !important;
}
.leaflet-control-zoom a {
  background: var(--card) !important;
  color: var(--foreground) !important;
  border-color: var(--border) !important;
}
.leaflet-control-zoom a:hover {
  background: var(--secondary) !important;
}
.leaflet-popup-content-wrapper {
  background: var(--card) !important;
  color: var(--card-foreground) !important;
  border: 1px solid var(--border) !important;
  border-radius: var(--radius) !important;
  box-shadow: 0 4px 24px oklch(0 0 0 / 0.4) !important;
}
.leaflet-popup-tip {
  background: var(--card) !important;
}
```

### Custom Pin Marker
Standard Leaflet default marker icon is broken with Vite (known issue). Replace with a custom SVG `DivIcon` using the amber accent color:

- SVG shape: teardrop/pin (16px wide, 22px tall)
- Fill: `oklch(0.80 0.14 78)` (primary amber)
- Stroke: `oklch(0.12 0.02 80)` (dark, 1px)
- Anchor: `[8, 22]` (tip of pin)

### Cluster Pin
A `DivIcon` with a 24px amber circle, dark count text (DM Sans 12px weight 400), amber border. Visual distinctiveness comes from the amber fill, not weight.

CSS: `background: oklch(0.80 0.14 78); color: oklch(0.12 0.02 80); border-radius: 50%; width: 24px; height: 24px; display: flex; align-items: center; justify-content: center; font-weight: 400; font-size: 12px; border: 2px solid oklch(0.12 0.02 80);`

### Layer Switcher
- Use Leaflet's built-in `L.control.layers` (not a custom React component)
- Position: `topright`
- Layer names: "Street" (OSM) and "Satellite" (Esri)
