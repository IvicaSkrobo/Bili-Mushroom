# Phase 05-01 Plan: Website Foundation

## Goal

Create the first Gljivobook website inside this repository so the app has a public home, bilingual messaging, download/support entry points, and a design foundation that can later pull live release/community data.

## Scope

- Add `/website` as a standalone static site workspace.
- English default route (`/`) and Croatian route (`/hr`).
- Pages/sections: Home, Download, Changelog, Support, Community, Ideas.
- Use Forest Codex visual identity and app screenshots.
- Keep release/community data as mocked or local JSON in this plan; live GitHub integration comes in 05-02 through 05-05.

## Tasks

1. Choose static stack and repo structure.
   - Preferred: Vite + React for consistency with the app, unless Astro is chosen before implementation.
   - Keep website dependencies isolated under `/website`.

2. Build bilingual information architecture.
   - `/` English home.
   - `/hr` Croatian home.
   - Keep language toggle visible and predictable.
   - English remains canonical default.

3. Create initial content.
   - Explain Gljivobook as a local-first mushroom field journal for Windows.
   - Highlight offline/local storage, collection organization, map, stats/export.
   - Add download and support CTAs.

4. Prepare media slots.
   - Hero app screenshot.
   - Collection screenshot.
   - Map screenshot.
   - Stats/PDF screenshot.
   - Use placeholders only if screenshots are not available.

5. Add deployment-ready build script.
   - Website can build independently.
   - GitHub Pages deployment comes in 05-02.

## Success Criteria

- Website builds locally without touching the Tauri app build.
- English and Croatian routes are reachable.
- Download, Support, Community, and Ideas entry points exist even if they still point to placeholder/manual URLs.
- Design matches Gljivobook instead of generic software landing-page styling.

## Validation

- Run website build.
- Check desktop and mobile responsive screenshots.
- Verify no horizontal scroll and no unreadable text in light/dark sections.
