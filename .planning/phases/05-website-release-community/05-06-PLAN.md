# Phase 05-06 Plan: Funding Goals

## Goal

Promote high-vote feature ideas into support/funding goals so users can see what their donations help build.

## Scope

- Funding goals on website.
- Manual funding progress in versioned JSON.
- Donate/support CTA per funded feature.
- Later-ready integration with donation provider APIs.

## Tasks

1. Choose support provider.
   - Candidates: Ko-fi, Buy Me a Coffee, PayPal, GitHub Sponsors, OpenCollective.

2. Add funding data model.
   - Feature title.
   - Linked idea discussion.
   - Vote count.
   - Goal amount.
   - Current amount.
   - Status.
   - Support URL.

3. Build funding UI.
   - Progress bar.
   - Current/goal amount.
   - Support button.
   - Clear note that support is optional.

4. Connect popular ideas to funding.
   - Ideas with enough votes can be manually promoted to Funding.
   - Released funded ideas move to Released section.

## Success Criteria

- Website shows funding goal cards.
- Each funding card links to its idea discussion and support URL.
- Funding progress can be updated without code changes beyond JSON.
- No payment-provider API is required for v1.

## Validation

- Add sample funding goal.
- Verify display in English and Croatian.
- Verify support link opens correctly.
