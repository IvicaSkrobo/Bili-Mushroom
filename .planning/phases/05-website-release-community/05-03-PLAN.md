# Phase 05-03 Plan: Real In-App Updater and Support Link

## Goal

Make the app's update and support flows work against real public infrastructure.

## Scope

- App update check/install against GitHub-hosted `latest.json`.
- Manual check remains visible in UI.
- Optional low-noise background recheck.
- Support/donate button opens external support page.

## Tasks

1. Verify updater configuration.
   - Stable app identifier remains unchanged.
   - Public key matches release signing key.
   - Endpoint points to actual `latest.json`.

2. Improve update UI states if needed.
   - Checking.
   - Up to date.
   - Update available.
   - Downloading.
   - Installing.
   - Failed with actionable message.

3. Add real support URL handling.
   - Centralize support URL.
   - Open in default browser via Tauri-safe method.
   - Keep support CTA subtle in Settings/About.

4. Link update UI to changelog.
   - When update is available, offer "View changelog".
   - Link opens website release/changelog page.

## Success Criteria

- App can detect a newer version from GitHub.
- App can install the update in a packaged Windows build.
- User can open support/donate page from Settings.
- Failure states are clear and not silent.

## Validation

- Test packaged old version against a newer release.
- Confirm update installs and app restarts.
- Confirm support link opens browser from packaged app.
