# Phase 05-05 Plan: Ideas and Voting Board

## Goal

Let users suggest features and let other users vote, so popular ideas can be prioritized or promoted to funding goals.

## Scope

- Idea submission through GitHub Discussions.
- Vote count from GitHub reactions.
- Website Ideas page sorted by votes.
- Status model for idea lifecycle.

## Tasks

1. Create idea discussion template.
   - Problem.
   - Proposed solution.
   - Why it helps.
   - Screenshots/examples optional.

2. Define statuses.
   - New.
   - Popular.
   - Planned.
   - Funding.
   - In progress.
   - Released.

3. Pull ideas into website.
   - Start with GitHub API or static generated JSON.
   - Sort by thumbs-up reaction count.
   - Show vote count, comment count, status, and link to discussion.

4. Add filters.
   - Popular.
   - Planned.
   - Funding.
   - Released.

## Success Criteria

- Users can submit feature ideas from website.
- Users can vote with GitHub reactions.
- Website lists ideas and sorts popular items first.
- Maintainer can mark idea status without a custom admin panel.

## Validation

- Create test idea discussion.
- Confirm vote count appears on website after rebuild/fetch.
