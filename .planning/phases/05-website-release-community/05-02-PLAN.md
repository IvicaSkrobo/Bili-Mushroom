# Phase 05-02 Plan: Release Automation

## Goal

Make GitHub Releases the source of truth for public downloads, changelog, and updater metadata.

## Scope

- GitHub Actions workflow for tagged releases.
- Windows Tauri build artifacts.
- Updater artifact generation/signing.
- `latest.json` upload.
- Website deployment/rebuild after releases.

## Tasks

1. Audit current release workflow.
   - Confirm tag trigger.
   - Confirm artifact names.
   - Confirm Tauri updater output and signing behavior.

2. Enforce version sync.
   - `package.json`
   - `src-tauri/tauri.conf.json`
   - `src-tauri/Cargo.toml`
   - app metadata used in UI

3. Build release artifacts on GitHub.
   - Windows installer.
   - Updater artifact.
   - Signature files.
   - `latest.json`.

4. Publish GitHub Release assets.
   - Installer visible for manual download.
   - Updater assets available at stable URLs.
   - Release notes used by website changelog.

5. Trigger website rebuild.
   - Website pulls latest release metadata at build time.
   - Download button updates automatically.

## Success Criteria

- Pushing a tag creates a GitHub Release with installer and updater assets.
- Website latest download points to that release.
- Changelog shows newest release notes.
- Failed version mismatch blocks release.

## Validation

- Dry-run release workflow if possible.
- Verify release asset URLs.
- Verify website build reads release metadata.
