# Phase 05-04 Plan: Community Layer

## Goal

Add a free, low-maintenance community layer where users can comment on releases, ask questions, report bugs, and discuss ideas.

## Scope

- GitHub Discussions categories.
- Giscus embed on website pages.
- Website links for Questions, Bugs, Ideas, and Releases.

## Tasks

1. Enable GitHub Discussions.
   - Categories: Announcements, Questions, Ideas, Bugs, Showcase.

2. Configure Giscus.
   - Use Discussions as backend.
   - Choose mapping strategy for changelog/release pages.
   - Add bilingual labels/copy around comment areas.

3. Add community entry points.
   - Ask a question.
   - Report a bug.
   - Suggest an idea.
   - Discuss latest release.

4. Add moderation guidance.
   - Short community rules.
   - Safety note: mushroom identification and edibility advice must not replace expert judgment.

## Success Criteria

- Users can comment through Giscus on release/changelog pages.
- Users can open GitHub Discussions from the website.
- No custom backend is required.

## Validation

- Test Giscus embed on deployed GitHub Pages.
- Confirm comments land in the expected Discussions category.
